VEHICLE

Vehicle is an ORM MVC DRS with REST on Express. Designed for real time responsive applications: Vehicle follows a paradigm similar to the classic MVC pattern (models, views, and controllers). However, it expands to include drivers, routes, and strategies to help build SaaS applications quickly using ORM and CRUD.

GETTING STARTED

Vehicle comes with an easy to use app generator. To get started open your console at and type in vehicle then press enter.

EXAMPLES

This vehicle test app comes with a few examples:

  • /hello_world - the most enthusiastic of examples ./controllers/hello_world.js
  • /test - loads the test controller from ./controller/test.js
  • /api - access the waterline CRUD api from ./controller/api.js
  • /api/users - access the users model from ./models/User.js via the driver ./drivers/User.js in Driver mode
  • /api/pets - inaccessable pets model from ./models/Pet.js via the driver ./drivers/Pets.js in Passenger mode
  • /template - the page you see here from ./controllers/template.js

DEVELOPERS

Vehicle is easy to learn and intuitive. If you already have an understand of the Model View Controller
paradigm and have had experience with nodejs we suggest you git clone the repo and check out
the app folders or generate a new vehicle app using the vehicle command prompt. Open said
app folder in your favorite file explorer. You will notice a very simple set of folders and
familiar files such as package.json and index.js. To get started open index.js.

Vehicle uses the following application folder structure:

./api - Used to store api keys
./assets - JS, CSS, Image assets
./configs - Vehicle configurations
./controllers - Controllers
./drivers - Drivers
./logs - Log directory for application level logging
./models - Models
./routes - Routes
./strategies - Strategies
./views - Application View Templates

EDITING

The core of your application will run from index.js within your app directory. This file
contains the main code operations for vehicle to run your app. Vehicle uses an optional
paradigm setup which allows you to pick and choose how your application modules respond
and interact with the vehicle core MVC. By default your app requires a folder named routes
and a file named wheel.js within. This file passes controller request route definitions to
controllers. Controllers can then optionally export data results back to vehicles view for output.

Models within vehicle also auto generate api access via CRUD. Models can be accessed by their
drivers designation. The driver in return loads the model collection and either returns data as
a driver or keeps to itself for processing as a passenger. Passenger data can not be accessed via
the /api route like driver data is. In Driver mode data is public and returned to the vehicle api
for CRUD consumption.
